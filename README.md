### Задание 1

Преимущества масштабирования с использованием конфигурации master-slave:
	- master-сервер обрабатывает все операции изменения данных, а slave-сервер — запросы на чтение. Это снижает нагрузку на master, позволяя ему эффективнее обрабатывать транзакции.
	- резервное копирование - slave-сервер действует как резервная копия данных, позволяет оперативно восстановить данные без остановки сервиса. Не является бэкапом!!!
	- повышение доступности данных  и базовая отказоустойчивость системы - slave хранит копию данных master-сервера и в случае сбоя можно быстро переключиться на slave (failover), минимизировав время простоя;
	- возможность обслуживания — поскольку slave-сервер содержит данные только для чтения, можно запускать процессы, обслуживающие базу данных, не прерывая работу системы. При необходимости можно использовать slave-серверы для запуска ресурсоёмких отчётов и аналитических запросов.


Преимущества масштабирования с использованием master-сервера и нескольких slave-серверов:
	 - распределение нагрузки - нагрузка от операций чтения распределяется между несколькими slave, это существенно повышает общую производительность системы в приложениях с интенсивным чтением данных.
	- увеличенная отказоустойчивость - наличие нескольких slave-серверов снижает риск потери данных при сбоях, при отказе одного slave-сервера нагрузка перераспределяется на другие;
	- балансировка распределение запросов - возможность реализации балансировкиу нагрузки между slave-серверами, настройка под конкретные задачи, кэширование данных и/или по географическим зонам(ускорение доступа пользователей). 
	- копии данных хранятся на нескольких серверах в разных локациях, что повышает надёжность резервного копирования и защиту от потери данных при локальных сбоях.
	- упрощение бэкапов и обслуживания - резервные копии можно снимать со slave-серверов, не нагружая master. Обслуживание и обновления slave-серверов можно проводить поэтапно, не останавливая всю систему

#### Важно отметить, что репликация не является резервным копированием. Случайное удаление данных на основном сервере приведёт к их удалению на всех репликах. Резервные копии необходимо создавать отдельно!
